## Overview

Fetches bot response time analytics stored in `CustomerSpecificData.botResTimeAnalytics` field of voice sessions. Supports date filtering and pagination.

---

## Query Parameters

<ParamField query="startDateTime" type="string">
  Start date/time for filtering (ISO 8601 format). Example: `2024-01-15T00:00:00Z`
</ParamField>

<ParamField query="endDateTime" type="string">
  End date/time for filtering (ISO 8601 format). Example: `2024-01-16T23:59:59Z`
</ParamField>

<ParamField query="skip" type="number" default="0">
  Number of records to skip for pagination
</ParamField>

<ParamField query="limit" type="number" default="1000">
  Maximum number of records to return
</ParamField>

## Field Descriptions

| Field | Type | Description |
|-------|------|-------------|
| `_id` | string | Voice session MongoDB ObjectId |
| `updatedAt` | string (ISO 8601) | Last update timestamp of the voice session |
| `botResTimeAnalytics` | object | Bot response time analytics data (structure varies by implementation) |

### Typical `botResTimeAnalytics` Fields

| Field | Type | Description |
|-------|------|-------------|
| `totalResponseTime` | number | Total response time across all bot turns (seconds) |
| `avgResponseTime` | number | Average response time per bot turn (seconds) |
| `maxResponseTime` | number | Maximum response time observed (seconds) |
| `minResponseTime` | number | Minimum response time observed (seconds) |
| `responseCount` | number | Total number of bot responses in the session |
| `details` | array | (Optional) Turn-by-turn breakdown with user input, bot response, and response time |

---

## Notes

- **Date Filtering:** Dates are adjusted to start/end of day in `Asia/Kolkata` timezone
- **Only Sessions with Analytics:** Returns only voice sessions that have `CustomerSpecificData.botResTimeAnalytics` field present
- **Pagination:** Use `skip` and `limit` for paginating large result sets
- **Default Limit:** 1000 records (can be adjusted via query param)
- **Sorted:** Results are sorted by `updatedAt` in descending order (most recent first)
- **Use Case:** Monitor bot performance, identify slow responses, optimize AI model latency
- The structure of `botResTimeAnalytics` may vary based on implementation; inspect actual responses for exact schema
