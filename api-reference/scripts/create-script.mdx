## Body Parameters

<ParamField body="script_name" type="string" required>
  Name of the script
</ParamField>

<ParamField body="script_description" type="string">
  Description of what this script does
</ParamField>

<ParamField body="script_content" type="object" required>

      Opening message for the conversation
    </ParamField>
    
    <ParamField body="system_prompt" type="string">
      System-level instructions for AI behavior
    </ParamField>
    
    <ParamField body="conversation_flow" type="array">
      Array of conversation steps/nodes
    </ParamField>
    
    <ParamField body="variables" type="object">
      Custom variables to use in the script (e.g., {name}, {company})
    </ParamField>
    
    <ParamField body="end_call_conditions" type="array">
      Conditions that trigger call termination
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="language" type="string" default="en-US">

<ParamField body="accent" type="string" default="neutral">

  **Options:** `neutral`, `american`, `british`, `australian`, `indian`, etc.
</ParamField>

<ParamField body="voice_settings" type="object">

    <ParamField body="speed" type="number" default="1.0">
      Speech speed multiplier (0.5 - 2.0)

</ParamField>

      Response creativity (0.0 - 1.0)

    <ParamField body="max_tokens" type="integer" default="150">

  </Expandable>

<ParamField body="call_settings" type="object">

## Script Variables

You can use these dynamic variables in your script:

| Variable | Description | Example |
|----------|-------------|---------|
| `{name}` | Contact's name | John Doe |
| `{company}` | Company name | Acme Corp |
| `{email}` | Contact's email | john@acme.com |
| `{phone}` | Contact's phone | +1234567890 |
| `{custom_field_name}` | Any custom field | Custom value |

## Best Practices

1. **Keep prompts concise** - Aim for 15-20 words per prompt
2. **Use natural language** - Write as you would speak
3. **Define clear end conditions** - Specify when to end the call
4. **Test with variables** - Ensure all variables are properly mapped
5. **Set appropriate timeouts** - Balance between responsiveness and patience
6. **Monitor AI temperature** - Lower (0.3-0.5) for consistent responses, higher (0.7-0.9) for creative

## Performance Notes

- **Script Creation**: Typically completes in < 1 second
- **Validation**: Complex scripts may take 2-3 seconds to validate
- **AI Model Loading**: First call with new model may add 5-10 seconds latency
- **Rate Limiting**: 50 script creations per hour
